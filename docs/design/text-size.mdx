저시력자는 스크린의 콘텐츠 사이즈를 늘리기 위해 브라우저 확대 도구와
[ZoomText](http://www.aisquared.com/products/zoomtext/)와 같은 보조 기술을 사용합니다.
이 연습에서는 모든 콘텐츠를 확대하는 것과
페이지의 텍스트만 확대한 것의 차이를 보여줍니다.

## 텍스트 크기

저시력자의 경우, 텍스트가 겹치거나 잘리지 않고 텍스트 크기를 늘리는 기능을 지원하는 것이 중요합니다.
테스트하려면,
브라우저의 내장 확대 또는 텍스트 크기 조정 기능을 사용하여 확대하고 텍스트를 계속 읽을 수 있는지 확인하세요.
모바일 웹을 구축 중인 경우, 핀치 줌을 비활성화하지 마세요.

[WCAG 지침은 콘텐츠 손실 없이 200%까지 확대를 허용할 것을 권장합니다.
](http://www.w3.org/TR/2008/REC-WCAG20-20081211/#visual-audio-contrast-scale)

[WebAIM 사이트에서 글꼴과 접근성에 대해 자세히 알아보세요.
](http://webaim.org/techniques/fonts/#font_size)

다음 연습의 경우, 전체 페이지를 확대하는 대신 텍스트 크기만 늘릴 수 있는
사파리 같은 브라우저가 필요할것입니다.
사파리에서, 보기로 가서 텍스트만 확대/축소로 이동한 다음 동일 메뉴에서 확대를 선택합니다.
다음 상자 밖으로 이 단어가 흘러 나오는 것을 알 수 있습니다.
너비와 높이를 em과 같은 상대 단위로 지정하여 이 예제를 수정해보세요.
높이를 3em과 너비를 10em으로 설정하고 답을 확인해보세요.

import { Editor } from "../_util/Editor.mdx";

<Editor
  checkExercise={(result) => {
    var container = document.querySelector("#container");
    const isValid =
      container.style.height === "3em" && container.style.width === "10em";
    if (!isValid) {
      return "It doesn't seem like the height and width are set to the right ems";
    }
  }}
>
  {`<div
  id="container"
  style="width: 80px; height: 50px;"
>
  Lorem ipsum dolor sit amet
</div>`}
</Editor>
