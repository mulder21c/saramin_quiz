스크린리더는 스크린의 텍스트와 상호작용 합니다. 따라서, 스크린리더 사용자에게 이미지의 의미를 전달하기 위해,
HTML에 접근 가능한 텍스트 레이블을 넣습니다. 이미지가 장식용인 경우, 빈 레이블(`alt=""`)을 제공하여 스크린리더 사용자에게 숨길 수 있습니다. 다음의 연습을 마친 후, [이 주제에 대한 WebAIM의 자료](http://webaim.org/techniques/alttext/)를 확인하여 이미지에 대한 좋은 대체 텍스트를 작성하는 것에 대해
더 많이 학습 할 수 있습니다.

## 접근 가능한 인라인 이미지

이것은 `alt` 어트리뷰트가 있는 이미지입니다. Mac 사용자라면 보이스오버를 켜고,
에디터 출력 섹션으로 탭하고 이미지를 탐색하도록 <kbd>Ctrl</kbd> + <kbd>Opt</kbd> + <kbd>Right Arrow</kbd>를
사용하세요. 보이스오버가 `alt` 텍스트를 낭독할 것입니다. 아래 두 번째 이미지의 경우,
`alt` 텍스트가 없기 때문에 보이스오버는 어떤 유의미한 것을 낭독하지 않을 것입니다.

<Editor>
  {`<img
  alt='Teach Access Bridge'
  src='/img/inline-image.png'
/>`}
</Editor>

## 접근 불가능한 인라인 이미지 연습

이것은 `alt` 텍스트가 없는 이미지 입니다. 보이스오버를 켜고 그것을 어떻게 읽는지 들어보세요. 이 이미지가 접근 가능해지도록 `alt` 어트리뷰트  "Teach Access Bridge"를 추가하고 보이스오버로 다시 테스트 해보세요.

<Editor
  checkExercise={(result) => {
    const images = Array.from(result.getElementsByTagName("img"));
    const missingAltTag = images.some((image) => !image.hasAttribute("alt"));
    if (missingAltTag) {
      return "It doesn't look like you added an alt to your image.";
    }
  }}
>
  {`<img
  src="/img/inline-image.png"
/>`}
</Editor>
